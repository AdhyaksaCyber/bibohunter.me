# Security Policy

## Supported Versions

Kami hanya memberikan update keamanan untuk versi berikut:

| Version | Supported        |
| ------- | ---------------- |
| 5.1.x   | Supported        |
| 5.0.x   | Not Supported    |
| 4.0.x   | Supported        |
| < 4.0   | Not Supported    |

Jika kamu menggunakan versi yang tidak didukung, harap upgrade untuk mendapatkan patch keamanan terbaru.

---

## Reporting a Vulnerability

Kami sangat menghargai laporan kerentanan keamanan dari komunitas.

### Cara Melapor
Kirim laporan ke:
- Email: security@yourdomain.com
- Atau via GitHub Issues (gunakan private report jika sensitif)

### Sertakan Informasi:
- Deskripsi vulnerability
- Langkah reproduksi
- Dampak (impact)
- Bukti (PoC atau screenshot jika ada)

---

## Response Time

- Respon awal: 1–3 hari
- Investigasi: 3–7 hari
- Patch (jika valid): secepat mungkin

---

## Responsible Disclosure

Kami meminta:
- Tidak melakukan eksploitasi berlebihan
- Tidak menyebarkan sebelum diperbaiki
- Tidak melakukan DDoS atau spam testing

---

## What to Expect

Jika laporan valid:
- Akan diperbaiki secepatnya
- Bisa mendapatkan credit (opsional)

Jika tidak valid:
- Tetap akan direview dan diberikan feedback

---

## Security Measures

Kami menerapkan:
- Firewall dan DDoS protection
- Rate limiting
- Input validation dan sanitization
- Secure headers (HSTS, CSP, dll)

---

## Disclaimer

Kami tidak bertanggung jawab atas aktivitas ilegal seperti:
- Hacking tanpa izin
- Data scraping berbahaya
- Eksploitasi sistem tanpa disclosure

Gunakan sistem ini secara bertanggung jawab.

---

## Warning

Segala aktivitas mencurigakan akan:
- Dicatat (logging)
- Diblokir otomatis
- Dilaporkan jika diperlukan
